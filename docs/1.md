Based on the project's source code, GridTokenX uses two different sets of features for its hybrid forecasting model (LightGBM and TCN).

Here is the comprehensive list of features used for model forecasting, categorized by the model that consumes them:

### 1. LightGBM Features (`models/lgbm_model.py`)

LightGBM is used for tabular/exogenous feature forecasting and relies on a wide array of engineered features:

**Weather Features (from TMD forecast, up to 72h ahead)**

- `Dry_Bulb_Temp`
- `Rel_Humidity`
- `Solar_Irradiance`
- `Heat_Index` (Engineered: Temp × Humidity)
- `Temp_Roll_Mean_3h`, `Temp_Roll_Mean_6h`
- `Humid_Roll_Mean_3h`, `Humid_Roll_Mean_6h`
- `Temp_Gradient`

**Market & Carbon Features**

- `Carbon_Intensity`
- `Market_Price`

**Calendar & Tourism Proxy Features**

- `Hour_of_Day`
- `Day_of_Week`
- `Is_Thai_Holiday`
- `Is_Songkran`
- `Tourist_Index`
- `Is_High_Season`

**Grid Dynamics (Stability & Capacity)**

- `Max_Capacity_MW`
- `Headroom_MW`

**Load History Features (from SCADA)**

- `Load_Lag_1h`, `Load_Lag_24h`, `Load_Lag_168h`
- `Load_Roll_Mean_3h`, `Load_Roll_Std_3h`
- `Load_Roll_Mean_6h`, `Load_Roll_Std_6h`

**Cluster Spatial Features (Neighboring Islands)**

- `Phangan_Load_Lag_1h`, `Phangan_Load_Roll_Mean_3h`, `Phangan_Load_Roll_Mean_6h`
- `Samui_Load_Lag_1h`, `Samui_Load_Roll_Mean_3h`, `Samui_Load_Roll_Mean_6h`

### 2. TCN Features (`models/tcn_model.py`)

The Temporal Convolutional Network (TCN) focuses on sequential load forecasting and uses a more streamlined set of features (`SEQ_FEATURES`):

**Load & Historical Lag**

- `Island_Load_MW` (The target series itself)
- `Load_Lag_1h`
- `Load_Lag_24h`

**Grid State**

- `BESS_SoC_Pct` (Battery State of Charge)
- `Headroom_MW`

**Weather**

- `Dry_Bulb_Temp`
- `Heat_Index`
- `Rel_Humidity`

**Calendar/Temporal**

- `Hour_of_Day`
- `Is_High_Season`
- `Is_Thai_Holiday`

_(Both of these models are blended together via a Ridge Meta-Learner to produce the final 24-hour target load `Island_Load_MW` forecast)._
